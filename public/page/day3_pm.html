Reading aloud

Let‚Äôs practice reading R scripts with a partner.

They‚Äôll listen to what you say and turn it into R code.

It‚Äôs like a game of telephone, but for R.


Instructions

  1. Find a partner.
    • One of you gets to be Bert and one of you is Ernie.
  2. Bert will look at the first code block and tell Ernie what it does.
  3. Ernie will then write some code that accomplishes what Bert said.
  4. Then Bert can offer more clues to help.
    • Try not to say the exact names of functions, like ‚Äúuse filter‚Äù or ‚Äúuse write_csv‚Äù
    • Instead, say what you want to accomplish, like "Drop the rows where..." or "Save the data as..."

The YAML

The YAML (‚ÄúYet Another Markup Language‚Äù) is the document settings stored atop the R Markdown document, bounded on each end by ---. When you first create a new .Rmd the YAML will look something like this:

---
title: "Document title"
author: "Author"
date: "MM/DD/YYYY"
output: html_document
---

Code chunk options

Chunk options are options designated in the code chunk header that determine what appears or does not appear in the final document.

For more information on code chunk options in R Markdown, see Chapter 11 in the R Markdown Cookbook.

Here are some common chunk options:

  • echo = FALSE: do not show the code in the knitted document
  • include = FALSE: do not include code or any outputs in knitted document
  • message = FALSE: suppress messages when knit
  • warning = FALSE: suppress warnings when knit
  • eval = FALSE: do not evaluate this code

Customization

Let‚Äôs make some changes to the YAML in our .Rmd to change the output document.

Let‚Äôs try the following:

  • toc: true: add a table of contents based on header hierarchies
  • toc_float: true: make it a floating TOC
  • number_sections: true: add numbered sections based on header hierarchy
  • theme: _____: add a bootstrap theme
  • code_folding: hide: code is default hidden, but available if the reader clicks on the ‚ÄòCode‚Äô button created


To add options to the YAML, add them as ‚Äúchildren‚Äù under the html_document: section.

Beware of spacing here. You will add 2 spaces of indentation for each new sub-level in the YAML hierarchy.

---
title: "Level up in R Markdown"
subtitle: "UCSB QMSS seminar series (February 2021)"
author: "Allison Horst"
output: 
  html_document:
    toc: true
    toc_float: true
    number_sections: true
    theme: flatly
    code_folding: hide
---

For more bootstrap themes visit: https://bootswatch.com/3/
